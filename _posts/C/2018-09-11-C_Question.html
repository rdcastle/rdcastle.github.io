배열 개념에서, 문자열도 결국 문자를 저장하는 배열이기 때문에,
배열명에 저장된 주소값과 배열의 첫 번째 인덱스에 해당하는 주소값을 각각 출력해 보았다. 근데.. 

      
        #include <iostream>
        #include <string>
        using namespace std;

        int main(void) {
          // 문자열 주소와 0번 인덱스 주소 출력
        	char ch[101];
        	char* compare = &ch[0];

        	cout << &ch << " == " << compare << endl;

          // 숫자 배열 주소와 0번 인덱스 주소 출력
        	int arr[101];
        	int* ptr = &arr[0];

        	cout << &arr << " == " << ptr << endl;

        	return 0;
        }

출력결과

      00BEF99C == a
      00BEF7F4 == 00BEF7F4
      계속하려면 아무 키나 누르십시오 . . .

숫자 배열의 경우 첫 번째 인덱스의 주소값은 그대로 출력이 되는데, 문자열의 경우 첫 번째 인덱스의 주소값이 출력되지 않고 그곳에 저장된 값이 출력되는가..?

아마 문자열의 특성상 아스터리스크(*) 없이 이름(주소값이 저장된)만으로도 저장된 값에 접근 가능하다는 것 때문이 아닐지..
(숫자 배열은 '*arr' 이런 식으로 저장된 값에 접근한다)
