HTML: 정보전달!
CSS: 디자인!

본래 CSS 라는 개념은 없었다. 디자인을 위해서 HTML의 font 와 같은 태그를 사용했을 뿐. 그러나 웹이 폭발적으로 성장하고 포함하는 정보의 양이 기하급수적으로 증가하면서, HTML 의 정보전달로서의 기능이 중요하게 되었다. 결국 문법을 더 명료하게 하고 데이터처리의 효율을 높이며 가독성 및 코드작성의 편의성을 위해 디자인을 위한 CSS 언어가 창조되었고, 그것은 HTML 로부터 완전히 분리되는 한편 그것과 함께 어우러져 보다 아름답고 간결한 웹 페이지를 작성할 수 있게 정교화되었다.

예를 들어 HTML로만 이루어진 만 줄짜리 코드에 수천 개의 font 태그가 산개되어 있다고 가정할 때, 그것들을 수정하거나 삭제하기 위해서는 엄청난 시간과 고된 작업이 필요하다. 그러나 그러한 공통적인 요소들을 한데 모아 하나의 연결된 문법으로 통일하면 하나를 수정함으로써 수많은 디자인 설정들을 변경하는 굉장한 효율을 발휘할 수 있다. 즉, HTML 이 고유의 의미가 있는 페이지 내 정보들을 담당한다면 CSS 는 고유한 의미는 없지만 그러한 정보들을 꾸며주는 별도의 기능을 전문적으로 담당한다고 생각하면 된다.

다만 CSS 에서 새롭게 발생한 고려사항은, 꾸미긴 꾸미되 무엇을 꾸밀것인가? 의 문제인데, 설령 같은 종류의 정보나 태그일지라도 작성자의 의도에 따라 더욱 한정된 대상들을 포착하는 것이 매우 중요한 이슈가 되었다. 그것이 바로 '선택자' 의 개념이다.

CSS 기본문법 - style 태그

HTML 태그와 분리된 CSS 는 자신만의 고유한 영역에서 독립적으로 작성된다. 그 영역은 head 태그 하위 style 태그 하위에 존재한다(style 태그까지는 HTML 문법이고, 그 하위공간이 CSS 영역). 이곳에서 무엇을 꾸밀 것인지 정확히 핀셋으로 집어내서(선택자) 각종 CSS 문법으로 디자인한다.

        <html>
          <head>
            <style>     << 여기서부터
              h2 {
                color:blue
              }
            </style>    << 여기까지
          </head>
        </html>
