Multi-Dimensional Array

대표적으로 2차원 배열을 생각해보면(그 이상은 잘 사용되지 않음), 그것은 행(row)과 열(column)로 구성된 일종의 테이블 형태라고 생각하면 된다.

  1. 선언 방식

    2. int arr[3][4];

    먼저 삽입된 인덱스값은 행(row)을 의미하고, 그 다음 것은 열(column)을 의미한다. 원하는 요소로 접근할 때는 정확한 행과 열의 인덱스를 입력해야 한다(ex. 3행 4열 접근 >> arr[2][3])

    각 인덱스값은 [0 ~ n-1] 로 이루어져 있음을 상기할 것!

  2. 실제 메모리 할당 형태

    3. 개념적으로는 행과 열로 이루어진 2차원 형태지만, 실제 메모리에 저장된 주소값을 확인해보면 1차원으로 구성된 것을 알 수 있다. 배열 선언 시 입력해준 row 또는 column 값에 근거하여 1차원적으로 나열된 각 배열공간의 값들을 2차원적으로 표현하는 것이다. (이때 근거로 삼기 위해 행, 열 중 최소한 하나의 값은 알려줘야 한다)

    예를 들어, 2행 3열짜리 2차원 배열에 1 ~ 6 까지의 수(int형)를 빠짐없이 저장한다면,

    0x1000 = 1
    0x1004 = 2
    0x1008 = 3
    0x100C = 4
    0x1000 = 5
    0x1004 = 6

    이런 식으로 실제 주소값에 연속적으로 저장된 후 2차원 인덱스에 맞게 호출된다!

  3. 배열 선언과 동시에 초기화

    4. int arr[3][4] = { {1, 2}, {3, 4, 5}, {6, 7, 8, 9} } or
    int arr[3][4] = { 1, 2, 0, 0, 3, 4, 5, 0, 6, 7, 8, 9 }

    위 두 방식은 모두 같은 위치에 같은 값이 저장된다
    row 와 column 을 기준으로 각 요소가 어떻게 구성되는지 충분히 인지한 후, 그것에 알맞게 값을 부여해야 한다. 행 기준으로 그룹화하여 입력하면, 빈 공간은 0 으로 채워진다

  4. 배열 출력

    5. 출력은 반복문을 이용하여 각 요소의 값을 규칙성 있게 호출하면 된다.

  5. 배열 크기를 모두 알려주지 않고 초기화

    6. 앞서 언급했듯이 최소한의 근거를 얻을 수 있도록 행과 열 중 적어도 하나의 인덱스값은 알려줘야 한다.

  6. 배열 크기 구하기

    7. sizeof() 연산자를 사용하면 된다
    sizeof(arr);
    단, 문자열의 경우 행 기준으로 그룹화되기 때문에(하나의 행에 속한 열은 모두 문자열의 구성요소) 전체 사이즈에 column의 크기를 나눠줘야 실질적으로 원하는 문자열의 값을 알 수 있다

+ 3차원 배열: int arr[3][4][3];
행(가로), 열(세로)에 높이의 개념이 추가된 것. 원리는 2차원 배열과 동일하다. 4차원 이상의 배열은 이론적으로는 가능하지만, 불필요하기도 하고 논리적으로 이해하기도 어려우므로 사용하지 않는다.
