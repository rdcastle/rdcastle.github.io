배열과 포인터

우선 열 칸짜리 배열을 선언하고

  • int arr[10] = { 11, 22, 33, 44, 55, 66, 77, 88, 99, 110};

배열명(주소값) 덧셈

배열명은 배열의 첫 번째 공간의 주소값을 갖는다(상수이기 때문에 변경이 불가하다).
위 배열 arr의 경우 자료형이 int 이므로, 첫 번째 공간의 주소값을 가진 배열명에 1을 더하면 실 주소값은 int 단위인 4Byte 씩 증가한다. 자료형이 int 라는 것은 배열 한 칸이 4Byte 라는 의미이기 때문이다.

  • 배열명은 주소값을 가진다!
  • 배열명은 상수이다! > 초기화 후 절대 못 바꾼다

포인터와 배열의 관계

배열명 외에 주소값을 갖는 또다른 변수가 있는데, 바로 포인터다. 포인터의 자료형은 'int*' 와 같은 방식으로 선언한다(* 는 주소값을 갖는다는 의미).
배열명의 덧셈과 마찬가지로 포인터 변수도 주소값을 가지므로, 해당되는 자료형의 크기에 따른 증감이 이루어진다(int 형이라면 4Byte 씩). 또한, 배열의 특정 인덱스에 접근하는 방식과 완전히 동일한 방식으로 포인터를 통한 인덱스 접근도 가능하다.
배열 arr 와 같은 주소값을 갖는 포인터를 p 라고 하면, 다음과 같이 접근하는 방식과 공간이 완전히 같다

  • arr[0] = 11, arr[1] = 22, arr[2] = 33, ... arr[9] = 110
  • p[0] = 11, p[1] = 22, p[2] = 33, ... p[9] = 110

여기서 도출할 수 있는 결론!!

  • 배열명과 포인터는 완전히 같은 개념이다!

왜 이런 결과가 도출되냐면, 컴퓨터 입장에서 배열명이나 포인터나 특정 주소값을 기반으로 간접접근한다는 사실은 같으며, 그것이 배열명인지 포인터인지 구분하는 것은 사실상 불가능하기 때문이다(전혀 인식하지 못한다, 사실 배열이니 포인터니 개념 자체가 없다. 그저 주소로 간접참조하는 것뿐).

배열과 포인터 간단정리

  • arr[0] = *(arr + 0) = *(p + 0) = 11 // arr[0] = *arr = *p
  • arr[1] = *(arr + 1) = *(p + 1) = 22
  • arr[2] = *(arr + 2) = *(p + 2) = 33
  • arr[3] = *(arr + 3) = *(p + 3) = 44
  • arr[4] = *(arr + 4) = *(p + 4) = 55
  • arr[5] = *(arr + 5) = *(p + 5) = 66
  • arr[6] = *(arr + 6) = *(p + 6) = 77
  • arr[7] = *(arr + 7) = *(p + 7) = 88
  • arr[8] = *(arr + 8) = *(p + 8) = 99
  • arr[9] = *(arr + 9) = *(p + 9) = 110

포인터와 배열의 차이점

지금까지 살펴본 바에 따르면, 배열명과 포인터는 완전히 같은 개념이다.
그러나 두 가지 미세한 차이점이 존재한다(경우에 따라 미세하지 않을 수도).

  1. 배열의 크기는 자유자재, 포인터의 크기는 무조건 4Byte!

    2. 사실 배열명 자체는 주소값만을 갖는다는 의미에서 사이즈가 자유자재라고 하기 어려운 부분도 있지만, 배열명이 가리키는 배열의 공간은 해당 자료형 단위로 가변적이다. 포인터 변수는 주소값을 갖는 4Byte 크기로 고정이다.

  2. 배열명은 상수, 포인터는 변수!

    3. 배열은 이미 할당된 공간이고, 배열명은 배열 공간만을 가리키도록 설계되어 있기 때문에 그 값이 변할 이유가 전혀 없다. 반면 포인터 변수는 설계상 주소값의 변화에 따라 다른 공간을 참조할 수 있다.
