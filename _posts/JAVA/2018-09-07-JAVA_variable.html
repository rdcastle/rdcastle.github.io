변수의 명명규칙

  • 대소문자가 구분되며 길이에 제한이 없다
  • 예약어를 사용해서는 안 된다
  • 숫자로 시작해서는 안 된다
  • 특수문자는 '_' 와 '$' 만 허용한다

필수는 아니지만 암묵적으로 권장되는 규칙

  • 클래스 이름의 첫 글자는 항상 대문자로 한다
  • 여러 단어로 이루어진 이름은 단어의 첫 글자를 대문자로 한다
  • 상수의 이름은 모두 대문자로 한다
  • 여러 단어로 이루어진 경우 '_' 로 구분한다

변수의 타입

  • 변수에 저장되는 값은 문자와 숫자로 나눌 수 있다
  • 실제 값(data)을 저장하는 기본형 변수와, 주소 값을 저장하는 참조형 변수로 나눌 수 있다
  • 기본형에 저장되는 실제 값의 형식을 자료형(data type)이라 하고, 참조형에 저장되는 참조하는 값의 자료형을 표현하는 보다 넓은 의미의 자료 형식을 타입(type)이라 한다
  • 클래스도 일종의 data type 이 될 수 있다 - data 가 저장되는 일련의 형식
  • 기본형에는 논리형(boolean), 문자형(char), 정수형(byte, short, int, long), 실수형(float, double) 이 있다
  • 각 data type 은 그것의 대표 사이즈에 따라 저장 가능한 값의 범주가 정해지며, 부호의 유무(signed, unsigned)에 따라 실질적인 표현범위가 제한된다
  • 변수의 자료형과 변수에 저장되는 수(일반적으로 상수)의 자료형이 일치해야 한다(명시)
  • 일치하지 않는다면 암묵적 변환규칙에 따른다(큰 사이즈가 작은 사이즈에 저장되면 데이터가 소실될 수 있음)

상수와 리터럴

  • 상수는 한번 저장된 값이 절대 변경되지 않는 수이며, 자바에서 상수지정은 final 키워드를 사용한다
  • 상수는 반드시 선언과 동시에 초기화해야 하며, 이름은 모두 대문자, 구분자는 '_' 이다
  • 리터럴(literal)은, 값을 저장하는 상수(constant)와 달리 그것 자체로 정해진 값을 의미하는 수이다
  • 리터럴이 있는데도 상수를 사용하는 이유는, 코드 내 같은 의미로 사용된 모든 리터럴을 유의미하게 일괄적으로 통제하기 위함이다
  • 일반적으로 리터럴은 변수에 저장될 때 사용되므로, 변수의 자료형과 일치되는 타입으로 지정되어야 한다
  • 디폴트 자료형은 정수는 int, 실수는 double 이지만, 리터럴 수에 접미사(L, f, d)를 붙임으로써 구체적으로 명시할 수 있다

타입의 불일치와 형변환

  • 큰 사이즈의 변수에 작은 사이즈의 데이터가 저장 가능하다
  • 작은 사이즈가 큰 사이즈에 저장되는 경우, double 형은 float 형에 저장불가, int 형은 short 나 byte 형에 저장 가능하다. 단, 변환하고자 하는 값의 표현범위 내여야 한다
  • 불일치하고 암묵적 형변환에 문제가 있다면, 데이터 손실을 무릅쓰고 명시적으로 형변환할 수 있다

    • 문자 리터럴과 문자열 리터럴

  • 문자 리터럴 - 'A' 와 같이 문자 하나를 작은따옴표로 감싼다, char 타입
  • 문자열 리터럴 - 'Java' 와 같이 두 문자 이상을 큰따옴표로 감싼다, string 문자(배)열, 이것은 빈 문자열로 사용 가능하다
  • String 은 클래스이므로 아래와 같이 객체를 생성하는 연산자 new를 사용해야 하지만, 연산자 오버로딩 기능에 의해 보다 간결한 표현 및 문자열 연산이 가능하다

    • String name = new String("Java");
    String name = "Ja" + "va";
    String str = name + 8.0;

    이때 더하기(+) 연산은 일반적인 숫자 끼리의 연산이 아니라, 적어도 하나 이상의 문자열 클래스 자료형을 포함하는 특수한 연산이다. 연산 오버로딩은 빈 문자열의 경우에도 적용된다.

    기본적인 함수(메서드)

    자바에서 메서드는 특정 클래스에 속하며, 메서드 호출을 위해서는 [클래스]. 를 통해야 한다.

    • println(); ==> System.out.println();, 줄바꿈 출력
    • printf(); ==> System.out.printf();, 보다 구체적인 출력(출력자료형 명시, 소수점 이하 자릿수 명시 등)을 위해 지시자(specifier)를 사용한다
    • Scanner 클래스

      • import java.util.*; ==> 클래스 사용을 위해 import java.util.*; 구문을 추가하고,
      • Scanner scanner = new Scanner(System.in); ==> Scanner 클래스 자료형 형식의 객체를 생성한다
      • 입력받은 내용을 문자열로 저장해서 정수형으로 변환 저장하기
        • String input = scanner.nextLine(); ==> scanner 객체 내 nextLine() 메서드(공백 포함 입력된 라인을 모두 읽음) 호출해서 입력받은 문자를 문자열(String) 변수 input에 저장한다
        • int num = Integer.parseInt(input); ==> 문자열을 int 타입으로 변환하기 위해 Integer 클래스의 parseInt(); 메서드를 호출한다(float 타입 변환은 Float.parseFloat(); 사용)
      • 정수로 입력받아서 바로 저장하기
        • int num = scanner.nextInt();
        • 같은 scanner 클래스의 nextInt(); 메서드를 호출하여 Integer 형식의 데이터를 입력받아,
        • int 형식으로 생성된 num 변수에 바로 저장한다
        • 그러나 이 메서드는 화면에서 연속적으로 값을 입력받아 사용하기에 까다롭다. 차라리 모든 값을 nextLine() 으로 입력받아서 적절히 변환하는 것이 더 낫다

    정수형의 오버플로우

    정수형과 실수형, 문자형까지 기본적으로 바이너리(이진) 데이터로 저장되므로, 자료형의 사이즈에 따라 표현 가능한 정보의 개수와 표현범위가 정해진다.
    오버플로우의 경우에도 이 표현범위를 벗어나면 발생하는 것이며, 이진수 연산원리에 따라 최대값에서 1이 증가하면 최소값으로, 최소값에서 -1 되면 최대값으로 전환된다.
