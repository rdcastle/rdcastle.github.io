Java 를 시작하기 전에

  • 자바란? - 96년 선 마이크로시스템즈에서 공식 발표한 객체지향 프로그래밍 언어!

    • 운영체제에 독립적, 인터넷 환경에 매우 적합, 객체지향언어 중 C++ 의 장점을 채택하고 단점이나 불필요한 점은 과감히 버림, 풍부한 클래스 라이브러리(Java API), 지속적인 성능 업데이트, 모바일(J2ME)이나 대규모 기업환경(J2EE), XML 등의 다양한 최신 기술 지원.

  • 자바의 역사

    • 본래 Oak 라는 이름으로 선의 엔지니어들에 의해 가전제품이나 PDA 같은 소형기기에 사용될 목적이었으나(객체지향), 다양한 환경이 통신 및 연결되는 인터넷이 등장하며 Java 로 변경되어 발표됨. 웹 보안상의 이유로 애플릿 프로그램 사용이 감소하는 대신 서버 쪽 프로그래밍을 위한 JSP 나 서블릿이 광범위하게 사용됨. 앞으로는 원래 목표였던 소규모 가전제품과 대규모 기업환경을 위한 SW 개발 분야에 활발히 사용될 것으로 기대됨.

  • 자바 언어의 특징

    1. 운영체제에 독립적 - Write once, run anywhere!, 가상머신인 JVM 사용
    2. 객체지향언어이다 - 상속, 재사용성, 캡슐화, 정보은닉, 다형성 등
    3. 비교적 배우기 쉽다
    4. 자동 메모리 관리 - Gargage Collector 로 사용되지 않는 메모리는 체크하여 회수한다. 자동적 메모리 관리기
    5. 네트워크와 분산처리를 지원한다 - 풍부하고 다양한 네트워크 프로그래밍 라이브러리(Java API)를 통해 비교적 짧은 시간에 네트워크 관련 프로그램 쉽게 개발 가능.
    6. 멀티쓰레드 지원 - 운영체제나 시스템에 관계 없이 멀티쓰레드 프로그램 구현 가능. 관련 라이브러리 활용 가능. 여러 쓰레드에 대한 스케줄링을 자바 인터프리터가 담당.
    7. 동적 로딩을 지원 - Dynamic Loading, 자바 프로그램 실행 시 모든 클래스가 로딩되지 않고 필요한 시점에 필요한 것만 로딩됨. 변경 사항에 대해 비교적 적은 작업으로 유연하게 처리

  • JVM(Java Virtual Machine)

    • 자바를 실행하기 위한 가상 기계. OS 와 SW 사이의 교량 역할(기계어 번역)을 하며, 적합한 JVM 이 설치된 컴퓨터라면 어떠한 Java 기반 프로그램도 컴파일 후 실행 가능하다

  • JDK(Java Development Kit)

    • 자바 프로그래밍을 위한 개발 도구. JVM 뿐만 아니라 JRE, 자바 클래스 라이브러리 등 필요한 모든 프로그램이 설치된다

    • 환경변수(PATH) 등록 - 일반적으로 JDK 는 C 드라이브에 설치되는데, c:\jdk1.8\bin 디렉토리(개발에 필요한 실행파일들이 들어 있음)의 경로를 환경변수에 등록하여 어느 공간에서든 경로명시 없이 이름만으로 필요한 파일 사용 가능하다.
    • bin dir 내 주요 실행파일 - javac.exe(자바 컴파일, 소스코드를 바이트코드로 컴파일), java.exe(자바 인터프리터, 생성된 바이트코드 해석 및 실행), javap.exe(역어셈블러)

      • ex) Hello.java --javac.exe--> Hello.class --java.exe--> 실행("Hello world" 출력!)

  • Java API Doc

    • 자바 API 문서에는 편리한 라이브러리에 대한 설명이 기록되어 있다. 모두 알기 보다는 기본적인 것만 숙지하고, 나머지는 필요에 의해 참조할 것.

  • 소스파일 및 클래스 명명하기

    1. 소스파일 하나 당 하나의 클래스를 정의하는 것이 일반적(둘 이상도 가능)
    2. 소스파일 하나 당 public 클래스는 하나만!(여러 개라면 각각 개별적인 소스파일에 저장할 것)
    3. 둘 이상의 클래스가 정의된 경우 public 제어가 적용된 클래스명으로 소스파일명 지정(public 이 없다면 아무거나 해도 됨)
    4. 파일이나 클래스, 기타 다양한 명명법은 영문위주로 작성하는 것이 좋고, 대소문자를 철저히 구분한다.

  • public static void main(String[] arge) { }

    • main: 메서드, 즉 함수명의 일종이며, 모든 자바 프로그램은 단 하나의 메인 메서드를 가져야 하며 이것의 호출과 종료로 시작과 끝을 맺는다(그 외 나머지 요소들은 기능 수행을 위한 부품들)
    • public: 메인메서드는 특성상 같은 프로그램 내 모든 클래스가 접근 가능(public)해야 한다
    • static: 또한 프로그램 실행의 시작과 끝을 의미하므로, 이것의 내용은 불변이다
    • void: 메인메서드의 종료는 프로그램의 종료를 의미하므로, 반환값이 필요 없으면 설령 있다 해도 아무런 의미가 없다
    • String[] args: 문자열 배열 선언이며, 'String args[]' 이런 방식으로도 선언 가능하다. 메인메서드가 호출될 때 어떤 값이 입력될지 알수 없으므로, 그러한 값들을 입력받기 위해 사이즈가 특정되지 않은 문자열 배열을 매개변수로 정의한다.
