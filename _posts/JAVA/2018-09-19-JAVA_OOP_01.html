객체지향언어 특성

  • 코드의 재사용성이 높다
  • 코드의 관리가 용이하다
  • 신뢰성이 높은 프로그래밍을 가능하게 한다

클래스와 객체

클래스는 설계도, 객체는 말 그대로 설계도를 토대로 실제로 존재하는 것을 의미한다. 객체는 사물과 같은 유형적인 것뿐만 아니라, 개념이나 논리와 같은 무형적인 것들도 포함된다

객체와 인스턴스

클래스로부터 객체를 만드는 과정을 클래스의 인스턴스화라고 하며 그러한 객체를 해당 클래스의 인스턴스라고 한다. 인스턴스는 사전적으로 '실제적인 사례' 정도의 의미를 갖는다

객체의 구성요소 - 속성과 기능

객체는 속성(preperty)과 기능(function, method)으로 구성되며 이들은 객체의 멤버라고도 한다

인스턴스의 생성과 사용

        [클래스명] [변수명];
        [변수명] = new [클래스명]();

이렇게 생성된 인스턴스에는 참조변수를 통해서만 접근이 가능하며, 참조변수의 타입은 인스턴스의 타입과 일치해야 한다(구조체와 같이 클래스도 일종의 자료형으로 본다, user-defined type)

객체 배열

객체들로 이루어진 배열을 정의할 수 있다. 길이가 3인 tv 클래스 배열을 선언하면 tv 객체 3개가 생성된 후 배열에 저장된다. 배열의 각 요소에는 별도의 공간에 실제 객체를 생성한 후 그곳의 주소값을 할당하여 잠조하도록 한다
