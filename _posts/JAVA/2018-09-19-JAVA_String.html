char 배열과 String 클래스의 차이점

문자를 연이어 늘어놓은 것이라는 의미에서 같지만, String 은 단순 배열이 아닌 클래스로서 유용한 기능(메서드)이 추가된 것이다

String 클래스의 주요 메서드

  • char charAt(int index): 문자열에서 해당 위치(index)에 있는 문자 반환
  • int length(): 문자열의 길이 반환
  • String substring(int from, int to): 해당 범위에 있는 문자열 반환(to 는 범위에 포함되지 않음)
  • boolean equals(String str): 문자열의 내용과 입력한 문자열이 같은지 확인. 결과는 boolean으로
  • char[] toCharArray(): 문자열을 문자배열(char[])로 변환해서 반환

문자열을 커맨드 라인을 통해 입력받기

커맨드 라인에서 실행할 프로그램의 main 메서드가 담긴 클래스의 이름 다음에 투입하고자 하는 문자열을 입력한다. 구분은 공백으로 한다
main 메서드의 매개변수로 사이즈가 정해지지 않은 String 배열 args 가 선언되어 있으므로, 넘겨받은 문자열이 몇 개가 되었든 저장 가능하다. 다만, 배열은 선언되었지만 아무런 값이 넘어오지 않은 경우 null 로 저장되어 에러가 발생될 것이지만, JVM 은 크기가 0 인 배열을 생성해서 자동적으로 args 에 전달하도록 구현되어 있으므로 문제가 없다

args 로 입력되는 데이터는 무조건 문자열로만 저장되기 때문에, main 함수 내부에서 원하는 자료형으로 변환하는 과정이 필요하다

  • int 로 변환: Integer.parseInt(args[0]);
  • char 로 변환: char op = args[1].charAt(0);
