input 태그

사용자 입력을 받아들이는 태그. 다양한 속성이 존재하며, 그 중 type 속성은 다양한 input 형태를 구현한다.

    • reset :

      • 자신이 속한 Form 태그 하위 모든 사용자입력사항 초기화

그 외에도 input 태그를 표현하는 다양한 속성들이 있다

  • name : 태그 이름 지정, 동일한 형태의 input 박스들 간의 특성과 의미를 구분하기 위해 의미 있게 지을 것(서버전송 시 name 으로 식별한다)
  • readonly : 읽기 전용으로
  • maxlength : 최대 글자 수 지정
  • required : 해당 태그가 필수태그로 지정됨. 이것을 입력하지 않고 submit 하면 에러메시지 출력됨(HTML5)
  • autofocus : 웹 페이지가 로딩되자마자 해당 태그로 포커스 이동(HTML5)
  • placeholder : 태그에 입력할 값에 대한 힌트(HTML5)
  • pattern : 정규표현식 사용, 유효한 값을 지정(HTML5)

label 태그

  • label : :

특정 input 태그와 관련된 의미론적인 설명을 연결시킴(설명을 위한 라벨을 붙이는 개념).
ID 라는 텍스트는 혼자서는 사실상 아무 의미도 없지만(브라우저에게 있어서), 이것에 'input 태그의 id 속성 값(id="id_txt")'과 연관된 label 태그를 붙여주면 'for="id_txt"' 두 태그는 id 속성을 매개로 연관성이 부여되어 의미 있게 컨트롤이 가능하다
위 input radio 타입 예제에서도 각 라디오 체크와 각 radio_1 ~ 3 은 상호간에 연관된 id 속성값으로 인해 의미론적으로 연결되어 있다(브라우저가 관련된 태그들로 인식 가능).

각각의 내용들을 input 혹은 label 태그로 묶어 id 로 연결하는 방식 외에, 관련된 모든 내용들이 연속적으로 존재한다면 아예 label 태그로 크게(한번에) 묶어주는 것도 하나의 방법이다 (이때는 사이에 관계없는 내용이 포함되지 않도록 주의해야 함)
