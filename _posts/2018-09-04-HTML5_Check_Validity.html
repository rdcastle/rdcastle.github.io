유효성 검사

<사용자가 잘못된 데이터를 입력하거나, 악의적인 코드를 입력하여 전송하고자 할 때 유효성 검사를 실시한다>

단순히 입력을 위한 텍스트박스를 만들고, 그것에 name 이나 id 속성을 부여해서 제어하는 방식보다는 앞서 살펴 본 새로운 속성값들과 같이 아예 type 속성 자체에 그것이 수용하는 데이터형식을 구체화하면 유효성 체크나 보안성 면에서 훨씬 효율적이다

    유효성 체크 방식

  • required 속성: 필수입력 유무를 결정한다
  • 정규표현식: 일종의 프로그래밍 언어로서, HTML과는 전혀 다른 것으로, 혼용해서 사용
  • pattern 속성: 정규표현식의 문법을 사용하여 가장 구체적인 사용자입력제한사항을 지정한다. 이것이 적용되면 정규표현식과 정확히 일치하는 데이터를 입력해야만 한다

    • ex) pattern="[a-zA-Z][a-zA-Z][0-9][0-9][0-9].."
    아이디 입력 시, 앞에서 두 자리는 알파벳으로, 다음 세자리는 숫자로, 그 이후 두 글자는 아무입력값이나 와도 되도록 제한

    '.' 은 아무 값이나 한 자리만 입력 가능하도록 하는 것이고, '.+' 는 아무 값이나 한 자리 이상 입력 가능하도록 하는 것

HTML에서 사용하는 정규표현식은 유효성 및 보안검사의 최소한의 수단이 되지만, 껍데기에서 입력되는 정보이기 때문에 특히 보안적인 측면에서는 굉장히 취약할 수 있다. 보안에서 HTML을 신뢰해서는 안 되고, 데이터를 전송받는 서버 측에서 주의깊게 접근제어를 해야 한다.
