변환생성자

객체가 생성될 때 오직 하나의 파라미터를 받아 멤버를 초기화하는 생성자

Conversion Constructor 는 매개변수의 타입에 따라 수용하는 인수를 암시적으로 형변환하는데, 예를 들어 Test 라는 클래스 형식의 User_Test 객체를 선언하고 생성자의 매개변수 타입을 int 라고 했을 때, 원래의 방식으로는 메인 함수에서 정해진 자료형의 인수를 Test User_Test(5) 처럼 넘겨줘야 하지만, 파라미터가 한 개만 정의되어 있는 변환생성자(Conversion Constructor)가 존재한다면, 변수에 대입하는 방식으로도 인수를 전달할 수 있다
Test User_Test = 5

대입 방식으로 전달되는 원리는(본래 자료형이 다른 int 와 Test 는 암시적으로도 호환될 수 없다), 클래스 데이터 타입(Test)과 다른 타입(int)의 데이터가 객체에 대입될 때 컴파일러는 int 형 데이터를 수용할 수 있는, 즉 int 형을 매개변수로 받는 클래스 내 생성자를 찾아 호출한다. 이것이 변환생성자이며 이 생성자는 int 형 데이터를 변환한 Test 형 객체를 임시로 만들어 대입하고자 하는 User_Test 객체에 대입한다

장단점

변환생성자는 객체를 선언하는 부분 외에도 언제든지 클래스형과 다른(변환생성자에 정의된) 자료형의 데이터를 객체의 멤버에 대입할 수 있다(초기화할 수 있다). 이는 굉장히 편리하고 효율적으로 멤버 초기화가 가능하도록 한다. 그러나 수용 가능한 모든 형식의 데이터를 무분별하게 받아들여 예기치 못한 버그를 발생시킬 수도 있다(의도치 않은 대입에 의한 객체 멤버초기화). 예를 들어 int 형의 경우 다양한 사이즈의 정수 및 실수, 심지어 문자형 데이터까지도 암시적으로 형변환되기 때문에 이것들이 실수로 객체에 대입되면 변환생성자에 의해 그대로 수용된다
